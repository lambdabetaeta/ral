# Type-enforced prevention for the cd-effect-leakage bug class

## Context

A bug in the zoxide plugin (`$env[PWD]` going stale across thunk calls) led
to two related root causes in `Shell::apply_chdir` and `Shell::return_to`:

- **Bug A (storage location)**: `apply_chdir` mirrored `PWD`/`OLDPWD`
  into `dynamic.env_vars` and the lexical scope. Both are roll-back-on-
  return scopes, while the process CWD is OS-global and persists. The
  cached env copy went stale on the first `cd` inside any thunk.
- **Bug B (forgotten flow-back)**: `Shell::return_to` flowed `plugin_context`
  back to the parent but not `pending_chpwd`, so `chpwd` never fired for
  `cd` inside thunks.

Both were patched manually (8 edits, tests pass). The user wants
**type-system-enforced prevention** of the bug class so a future change
cannot reintroduce either half.

The four state-relationship categories between parent and child shells
during a thunk call (plain-English names already agreed):

- **Bound** — child sees a clone, child's writes drop on return. (`env`, `dynamic`)
- **Returned** — child starts fresh, child's value flows back. (`pending_chpwd`, `last_status`)
- **Borrowed** — parent lends to child, child returns at end. (`plugin_context`, `audit.tree`)
- **Ambient** — outside the call entirely; OS-owned. (process CWD, fds)

## Approach

Three separate type-level mechanisms, each closing one half of the bug class:

1. **`EnvKey` newtype** with private constructor that refuses `"PWD"` / `"OLDPWD"`. `EnvOverrides` keys are `EnvKey`, so process-owned strings cannot enter the roll-back-on-return store. *Closes Bug A.*

2. **`Borrowed<T>` and `Returned<T>` wrappers** carry their own `lend_from` / `return_to` methods. Forgetting to call them in `Shell::return_to` is a missing-method-call, not a missing-line-in-prose. *Closes Bug B for the fields that get the wrappers.*

3. **`Shell::cwd()` accessor + clippy lint** banning `std::env::current_dir` outside it (mirrors the existing lint on `fs::canonicalize` in `clippy.toml`). *Enforces the Ambient discipline.*

`Bound` and `Ambient` need no wrapper types — `Bound` is just `T: Clone` with the existing inherit/return discipline; `Ambient` is "no field at all" enforced by the lint. The four categories live as **doc-comment vocabulary** in `core/src/types/flow.rs`.

Scope is bounded: refactor `dynamic.env_vars`, `plugin_context`, `pending_chpwd`. Leave `audit.tree`, `last_status`, `registry`, `modules`, `io` alone — too many callsites for marginal benefit, and their disciplines are working.

## Changes

### 1. `core/src/types/flow.rs` (new)

Module doc explains the four categories with examples. Defines:

```rust
pub struct Borrowed<T>(Option<T>);

impl<T> Borrowed<T> {
    pub fn empty() -> Self;                               // Default
    pub fn from(value: T) -> Self;                        // for direct init
    pub fn as_ref(&self) -> Option<&T>;
    pub fn as_mut(&mut self) -> Option<&mut T>;
    pub fn is_some(&self) -> bool;
    pub fn replace(&mut self, value: Option<T>);          // for fresh assignment
    pub(crate) fn lend_from(&mut self, parent: &mut Self); // call entry
    pub(crate) fn return_to(&mut self, parent: &mut Self); // call return
}

pub struct Returned<T>(Option<T>);  // hardcoded take-if-some semantics

impl<T> Returned<T> {
    pub fn empty() -> Self;
    pub fn set(&mut self, value: T);
    pub fn take(&mut self) -> Option<T>;
    pub fn is_some(&self) -> bool;
    pub fn as_ref(&self) -> Option<&T>;
    pub(crate) fn return_to(&mut self, parent: &mut Self); // take-if-some
}
```

Crucially: no `take()` on `Borrowed` (only `as_mut()` for in-place mutation). The take/restore discipline is the only way the value moves; there is no public method that lets you walk away with the inner value. `replace(Some(_))` exists for fresh assignments at clear lifecycle points (keybinding handler entry, plugin hook entry).

### 2. `core/src/types/env_overrides.rs` (new)

```rust
#[derive(Debug, Clone, Hash, Eq, PartialEq)]
pub struct EnvKey(String);   // private field — only constructor checks

#[derive(Debug)]
pub struct ProcessOwnedKey(pub String);

impl EnvKey {
    pub fn try_new(s: impl Into<String>) -> Result<Self, ProcessOwnedKey>;
    pub fn as_str(&self) -> &str;
}

#[derive(Debug, Default, Clone)]
pub struct EnvOverrides(HashMap<EnvKey, String>);

impl EnvOverrides {
    pub fn insert(&mut self, k: EnvKey, v: String) -> Option<String>;
    pub fn get(&self, k: &str) -> Option<&str>;
    pub fn contains_key(&self, k: &str) -> bool;
    pub fn remove(&mut self, k: &str) -> Option<String>;
    pub fn iter(&self) -> impl Iterator<Item = (&str, &str)>;
    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;
    pub fn extend_from_strings<I>(&mut self, items: I)
        where I: IntoIterator<Item = (String, String)>;  // silently skips PWD/OLDPWD
    pub fn entry_or_insert(&mut self, k: EnvKey, v: String);  // .entry().or_insert pattern
}
```

`extend_from_strings` deliberately drops process-owned keys (used by IPC wire deserialization where the source has already been through string-land).

Module-private: tuple field of `EnvKey`. Construction outside the module *cannot* produce `EnvKey("PWD")`.

### 6. `Shell::cwd()` accessor

In `core/src/types/shell.rs`, add:
```rust
/// Live process working directory.  Never cached — there is one cwd
/// in the world, owned by the OS.  Reading it here, not through a
/// stored field, is what makes Ambient discipline unforgeable.
pub fn cwd(&self) -> std::path::PathBuf {
    #[allow(clippy::disallowed_methods)]
    std::env::current_dir().unwrap_or_default()
}
```

Replace direct `std::env::current_dir()` calls outside `apply_chdir` and `seed_default_env` with `shell.cwd()` where `shell` is in scope.

### 7. `clippy.toml` lint

Add to the existing `disallowed-methods` list:
```toml
{ path = "std::env::current_dir", reason = "Use Shell::cwd() — process CWD must not be cached in any shell-scoped store; that's how PWD went stale." },
```

Permit only inside `Shell::cwd()`, `Shell::apply_chdir`, and `seed_default_env` (process-env initialization in `ral/src/platform.rs` and `exarch/src/eval.rs`) via `#[allow(clippy::disallowed_methods)]`.

### 8. Update doc on `core/src/types/repl.rs`

The "Flow rules" section becomes terser — the type itself encodes the rules. The doc just says "see flow.rs."
